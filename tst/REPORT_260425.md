## 2. Session reorganisation (2026-04-24)

The original monolithic `GeoTop.thy` was split into three theories
across three Isabelle sessions, with a fourth (`Top0`) holding the
general-topology background:

```
Top0          (i/)         general topology, AlgTop
GeoTopBase0   (b0/)        Moise Intro: simplexes, complexes,
                           subdivisions, barycentric infrastructure,
                           Euclidean topology bridges            (~9 962 lines)
GeoTopBase    (b/)         PL maps, isomorphisms, PLH, cells,
                           manifolds, §1–3                       (~12 460 lines)
GeoTop        (./)         §§4–36                                (~12 864 lines)
```

The split point was chosen to put the **Moise Introduction infrastructure
in a stable cached lower layer**. Once `GeoTopBase0` is built and
cached, all subsequent edits in `GeoTopBase` or `GeoTop` see only the
upper sessions rebuild. This yields ~13 s saved per iteration during
work on §1 and beyond. The split is documented in
`memory/project_geotopbase_split.md`.

Side-effect noted in the split: a small number of pre-existing `by100`
calls in `GeoTopBase0` became more visibly load-sensitive (their
budget is tight), occasionally tripping under parallel load. The
agreed remedy is `-o threads=1` for stable build timing during
development, with the default parallel build used for verification.

The methodology established in the previous report (`sorry`-first
proof scaffolding, `by100`-guarded automation, no-global-simp

practices added during this engagement:

### 4.1 Memory file index

A 39-entry index `MEMORY.md` now points to per-topic memory files
under `~/.claude/projects/-project/memory/`. Each entry is one line
with a one-sentence hook. Topics range from durable feedback

to active project state (`project_geotop_carrier_lemma_done.md`,
`project_geotop_moise45_done.md`). The memory index is consulted
at the start of each session, allowing fast resumption.

### 4.2 The `by100` failure-mode catalogue

A list of `by100` patterns that look natural but fail under the 100 ms
budget has been compiled (`feedback_by100_no_modifiers.md`,
`feedback_scaler_cancel_in_simp.md`, `feedback_geotop_blast_metis_bridge.md`).
Examples:

* `by (by100 (simp add: foo bar))` — modifiers not parsed; rewrite
  with `unfolding` or `using lem` chains.
* `by100 simp` on goals containing `α *⇩R v = β *⇩R v` (`scaleR_cancel_right`
  rewrites to the disjunction `α = β ∨ v = 0`, blocking subsequent
  `sum.cong`).
* `by (by100 blast)` on quantifier-rich existentials with three or more
  unknowns; `apply (rule exI[of _ witness])` chains are required.

These patterns are now caught during code review rather than at build
time.

### 4.3 The "fix-the-statement" reflex

Several lemmas this engagement turned out to be **mathematically false
as stated**, not merely hard to prove. The protocol for these
discoveries is:

1. Construct an explicit counterexample.
2. Add an in-source `WARNING` block citing the counterexample.
3. Update `MEMORY.md` and `PLAN1.md`.
4. Document the structural fix needed.
5. Leave the existing sorry in place (so the bug propagates upstream
   visibly) until a refactor session.

The two such bugs identified in this engagement are
`geotop_convex_in_complex_in_simplex` (this report's headline
remaining sorry, see §7) and the earlier-fixed
`geotop_bary_lin_inj_preserves_ai` and `Sd` spec (carried over from
the previous report).

## 6. The Moise 4.5 closure (today's headline)

`geotop_flag_intersect_hull_sub`, the chain-simplex intersection
equality (Moise Lemma 4.5, the K.2 axiom hard direction for the
barycentric subdivision), is now **fully proven**. This is the most
technically demanding theorem in the K-axiom verification of `bK`.

### 6.1 Statement

```
lemma geotop_flag_intersect_hull_sub:
  fixes K :: "'a::euclidean_space set set"
  assumes hK: "geotop_is_complex K"
  assumes hc1: "c1 ∈ geotop_flags K"
  assumes hc2: "c2 ∈ geotop_flags K"
  shows
    "convex hull (geotop_barycenter ` set c1)
       ∩ convex hull (geotop_barycenter ` set c2)
       ⊆ convex hull (geotop_barycenter ` (set c1 ∩ set c2))"
```

### 6.2 Proof structure

Strong induction on `card (set c1 ∪ set c2)` via `less_induct` with
`arbitrary: c1 c2`. The proof comprises ~700 lines and three nested
case splits.

* **Edge cases (D₁ = ∅ or D₂ = ∅):** support of α (or β) is `{σ₁}`,
  giving `α σ₁ = 1` (via `sum.remove` + `sum.neutral`), hence
  `x = bary σ₁`. Since σ₁ ∈ set c₁ ∩ set c₂, x is in
  `convex hull (bary ` (set c1 ∩ set c2))` via `hull_subset`.
* **Inductive case (D₁, D₂ both non-empty):**
  1. Construct sorted sub-flags `d₁ = filter c1` by D₁-membership and
     `d₂` analogously. Verify they are flags via
     `sorted_wrt_filter` and `distinct_filter`.
  2. Cardinality reduction `card (set d1 ∪ set d2) <
     card (set c1 ∪ set c2)` via `card_Diff1_less` (σ₁ excluded).
  3. Apply IH: `T_{d1} ∩ T_{d2} ⊆ T_{d1 ∩ d2}`.
  4. Set up `Polytope.convex_hull_insert_Int_eq` with
     `z = bary σ1`, `S = σ1`. The required hypotheses
     `T, U ⊆ rel_frontier σ1` come from a new helper
     `geotop_chain_hull_in_rel_frontier` that uses K.2 of K and
     `face_of_subset_rel_frontier`.
  5. Show `x ∈ conv hull (insert z T)` and
     `x ∈ conv hull (insert z U)` by constructing explicit
     u-functions on `insert z (bary ` set d_i)`, using bary
     injectivity on K-simplices, and applying `convex_hull_finite`.
  6. Combine via `h_insert_Int_eq` to get
     `x ∈ conv hull (insert z (T ∩ U))`.
  7. Chain through the IH and use σ₁ ∈ set c₁ ∩ set c₂ plus
     `hull_minimal` with target convex to get
     `x ∈ conv hull (bary ` (set c1 ∩ set c2))`.

### 6.3 Supporting infrastructure (eight new lemmas, ~1 280 lines)

The proof rests on a layered tower of helpers built across this
engagement:

| Lemma                                              | Purpose                          | Lines |
| ---                                                | ---                              | ---:  |
| `geotop_chain_vertex_subset`                       | V(s) ⊆ V(t) for s ⊆ t in K       |   75  |
| `geotop_bary_in_rel_interior_support`              | AI bary combo → rel_interior     |   60  |
| `geotop_chain_bary_rel_interior` *(carrier lemma)* | x = Σ β·bary σ → x ∈ rel_int σ_max | 250 |
| `geotop_chain_support_max`                         | Chain-top extraction             |   90  |
| `geotop_in_T_chain_to_alpha`                       | x ∈ T_c → α coefficients         |   60  |
| `geotop_chain_hull_in_rel_frontier`                | Sub-chain hull in rel_frontier   |   75  |
| `geotop_flag_intersect_hull_sub` *(Moise 4.5)*     | Chain-simplex intersection       |  700  |
| `geotop_complex_polyhedron_point_carrier`          | K-carrier existence              |  150  |
| `geotop_subdivision_simplex_in_parent`             | K' simplex ⊆ K simplex           |   12  |
| `geotop_iterated_Sd_simplex_in_K_simplex`          | Sd^m simplex ⊆ K simplex         |   10  |

The carrier lemma `geotop_chain_bary_rel_interior` deserves special
mention. Its proof (~250 lines) extracts `V_max =
simplex_vertices σ_max`, defines a `Vof` selector for each chain
simplex via `SOME`, expands `bary τ = (1/|V_τ|) Σ V_τ`, and produces
new bary coordinates `α v = sum over chain of β/|V_σ|`. The double
sums require `sum.swap`, `sum.If_cases`, and explicit case
splits between β = 0 and β > 0; the entire chain
`x = Σ α v *⇩R v` then matches `rel_interior_convex_hull_explicit`.

This carrier lemma alone closes seven theorems downstream that had
been deferred to "Phase 3 deep classical blocker" in the previous
report.

## 7. The remaining buggy lemma

The single sorry remaining in `GeoTopBase0.thy` is for
`geotop_convex_in_complex_in_simplex`:

```
lemma geotop_convex_in_complex_in_simplex:
  fixes K :: "'a::euclidean_space set set"
  assumes hK: "geotop_is_complex K"
  assumes hKfin: "finite K"
  assumes hT_conv: "convex T"
  assumes hT_ne: "T ≠ {}"
  assumes hT_sub: "T ⊆ geotop_polyhedron K"
  shows "∃σ∈K. T ⊆ σ"
```

### 7.1 The bug

This lemma is **provably false as stated**. Counterexample:

> Let `K` be a 2-triangle simplicial complex `{T₁, T₂, e, …}` where
> `T₁`, `T₂` share edge `e`. Let `T` be a small convex disk
> centred on a vertex of `e`. Then `T` is convex, non-empty,
> contained in `|K|`, but spans the rel-interiors of `T₁`, `T₂`,
> and `e` simultaneously. Therefore `T ⊆ σ` fails for every
> `σ ∈ K`.

The previous report's `PLAN1.md` proof sketch — "convex set's
K-carriers form a chain" — is incorrect: non-comparable simplices
that share a face can each contain points of `T`.

### 7.2 The bug propagation chain

The buggy lemma underpins a four-link proof chain:

```
geotop_convex_in_complex_in_simplex   (this lemma, FALSE)
  →  h_star_to_simplex_del            (also false as stated)
  →  h_δ_ex                           (Lebesgue-style, depends above)
  →  iterated_Sd_refines_subdivision  (Moise Lemma 4.17)
  →  Theorem_GT_1                     (Moise's first theorem)
```

Because the buggy lemma carries a `sorry`, the upstream proofs
*formally* compile, but their content is unverified. The bug needs
to be cleared before `Theorem_GT_1` can be considered proved.

### 7.3 The structural fix needed

The classical Munkres argument for `iterated_Sd_refines_subdivision`
does *not* go through a "convex T fits in single simplex" lemma.
Instead it uses:

1. Each `Sd^m K` simplex `τ` has a unique **K-carrier** — the
   smallest K-simplex containing `τ`, derived from the K-flag whose
   barycenters constitute τ's vertices.
2. For `K'` a subdivision of `K`, the K-carrier of `τ` is itself a
   union of `K'`-simplices, and small enough `τ` (mesh < δ) fits in
   one specific K'-piece via Lebesgue applied to the K'-subdivision
   of that K-simplex.

This bypasses the convex-in-complex strategy entirely. The
infrastructure required is:

* The K-carrier existence lemma `geotop_complex_polyhedron_point_carrier`
  (added today, fully proved).
* Two corollaries `geotop_subdivision_simplex_in_parent` and
  `geotop_iterated_Sd_simplex_in_K_simplex` (added today, fully
  proved).
* A new Lebesgue-on-K-simplex argument: for each `σ ∈ K`, the
  finite cover by `K'`-simplices contained in `σ` gives a δ_σ; take
  the min over `σ ∈ K`.

The carrier-existence lemma plus the two corollaries provide the
foundation. The remaining piece is the per-simplex Lebesgue
argument and the rewriting of `iterated_Sd_refines_subdivision` to
not go through `h_δ_ex`. Estimated ~300 lines; this is the
multi-session task identified for the next engagement.
